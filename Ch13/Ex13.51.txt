Question: 13.51: Although unique_ptrs cannot be copied, in ยง 12.1.5 (p. 471) we wrote a clone function that returned a unique_ptr by value. Explain why that function is legal and how it works.

Code:

unique_ptr<int> clone(int p) {
    // ok: explicitly create a unique_ptr<int> from int*
    return unique_ptr<int>(new int(p));
}

unique_ptr<int> clone(int p) {
    unique_ptr<int> ret(new int (p));
    // . . .
    return ret;
}

Answer:
The function is legal as it returns a rvalue, clone does not copy the object the unique_ptr points to, but moves it to the new unique_ptr being returned.
